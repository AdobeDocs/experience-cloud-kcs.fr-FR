# Aucune taille d’audience dans la destination AEP Microsoft Bing

## Description {#description}

Pourquoi Bing n’affiche-t-il aucune taille d’audience pour les segments AEP envoyés à la destination Bing AEP ?

## Résolution {#resolution}


Les trois raisons les plus courantes pour lesquelles une destination AEP Bing n’entraîne aucune taille d’audience sont les suivantes :

- L’ID de client Bing Ads est incorrect.
- La synchronisation des identifiants n’a pas été activée.
- La synchronisation des identifiants ne se déclenche pas sur le site web


Le [documentation de cette intégration](https://experienceleague.adobe.com/docs/experience-platform/destinations/catalog/advertising/bing.html?lang=en) indique que ce dont vous avez besoin pour configurer cette destination est l’ID de client Bing Ads. Il s’agit d’une <u>entier</u> et <b>ne sera pas </b>contiennent toutes les lettres. Ce morceau de [Documentation Bing](https://learn.microsoft.com/en-us/advertising/guides/get-started?view=bingads-13) vous aidera à trouver l’ID de client. La section &quot;Obtention de votre compte et de vos ID de client&quot; vous aidera à déterminer votre ID de compte par rapport à votre ID de compte. Malheureusement, l’API Bing ne répond avec aucune erreur pour indiquer que le mauvais ID a été utilisé. Les accès sont toujours envoyés à et acceptés par Bing, même si l’ID est incorrect. Si vous êtes un client AAM avec la version AAM de cette destination en place (et qu’elle fonctionne), il s’agit du même ID de client Bing Ads utilisé dans la destination d’Audience Manager.

La deuxième raison répertoriée s’applique uniquement aux clients qui n’ont pas déjà configuré et utilisé une version AAM de cette destination. Identique [document de configuration](https://experienceleague.adobe.com/docs/experience-platform/destinations/catalog/advertising/bing.html?lang=en) indique également la nécessité de synchroniser les identifiants si vous ne l’avez pas déjà configuré pour que AAM commence. Créez un cas de prise en charge pour que cette synchronisation des identifiants soit activée si vous ne l’avez pas déjà fait.

Enfin, même si la synchronisation des identifiants a été activée, elle doit se déclencher sur le site web en question. A l’aide d’un navigateur incognito pour imiter la première page d’une première visite, visitez le site web en question et utilisez les outils de développement de votre navigateur pour vous assurer qu’un appel a été effectué à cette URL :

https://c.bing.com/c.gif?uid=%DID%&amp;Red3=MSAdobe_pd&amp;gdpr=%GDPR_APPLIES%&amp;gdpr_consent=%GDPR_CONSENT% (les % seront remplacés par les valeurs réelles)

Si cet appel réseau ne se déclenche pas, la synchronisation des identifiants n’a pas lieu. Pour les utilisateurs du service d’ID Experience Cloud, cela peut signifier que[Les synchronisations D étaient désactivées.](https://experienceleague.adobe.com/docs/id-service/using/id-service-api/configurations/disableidsync.html?lang=en). Pour les utilisateurs de WebSDK, cela signifie probablement que les synchronisations des identifiants tiers n’étaient pas activées. Pour les activer, voir [article](https://experienceleague.adobe.com/docs/experience-cloud-kcs/kbarticles/KA-20248.html?lang=fr).
