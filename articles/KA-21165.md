# Campaign Classic : &quot;Messages mis en file d’attente (contrôle de flux dynamique)&quot; est rencontré lors de la diffusion de l’email

## Description {#description}


Le client a rassemblé les messages d’erreur pour les domaines les plus courants avec lesquels nous rencontrons des problèmes de délivrabilité pour ses emails client. Le volume le plus élevé est associé à l&#39;erreur &quot;Messages en file d&#39;attente (contrôle de flux dynamique)&quot;


## Résolution {#resolution}


1. Les erreurs de &quot;contrôle de flux dynamique&quot; se produisent lorsque le MTA comporte trop d&#39;erreurs dans un domaine en une courte période. Le MTA est codé pour entrer dans ce mode &quot;backoff&quot; (backoff) afin d&#39;empêcher l&#39;aggravation d&#39;un événement de blocage/ralentissement dans un domaine, ce qui pourrait très bien se produire s&#39;il ne ralentit pas ou n&#39;arrête pas l&#39;envoi temporairement.
2. Et pour être clair, le terme &quot;erreurs&quot; peut signifier toute tentative échouée de connexion ou de diffusion vers un domaine. Je veux dire par là, il pourrait s&#39;agir d&#39;une erreur MTA interne, pas seulement d&#39;un rebond SMTP ou d&#39;un refus de connexion du domaine. Ainsi, le client devra enquêter là-dessus de votre côté et s&#39;assurer que vous n&#39;avez pas de problèmes avec le MTA lui-même.
3. Supposons que les erreurs de &quot;contrôle dynamique de flux&quot; se produisent dans quelques domaines seulement. Dans ce cas, il est plus probable que cela soit dû à la réputation de délivrabilité, et qu’ils soient bloqués/limités par ces domaines. Et s&#39;il s&#39;agit d&#39;un problème de réputation de délivrabilité, le client aura besoin d&#39;heures de délivrabilité pour que quelqu&#39;un l&#39;aide à enquêter sur la cause et à déterminer une stratégie pour résoudre les problèmes.
