# Comment optimiser les index Oak Lucene pour réduire le temps d’indexation

## Description


Optimisez les index AEM Oak lucene pour réduire le temps de réindexation.
<br><br>Environnement<br><br>
AEM 6.0 + Oak 1.0.16 ou version ultérieure AEM 6.1 et 6.2


## Résolution


Vous pouvez optimiser les index Oak Lucene en affinant le paramètre oak: QueryIndexDefinition avec des propriétés telles que &quot;includedPaths&quot;, &quot;excludedPaths&quot;, &quot;name&quot; et &quot;indexPath&quot;.

- Les propriétés &quot;includedPaths&quot; et &quot;excludedPaths&quot; limitent les chemins parcourus par l’index lors de la réindexation et les chemins qu’il écoute pour les mises à jour de l’index.
- Les propriétés &quot;name&quot; et &quot;indexPath&quot; facilitent la compréhension des journaux d’index, car elles baliseraient les messages du journal avec le chemin ou le nom de l’index.  Par exemple:



| `30.01.2017 14:07:12.934 *DEBUG* pool-8-thread-2 org.apache.jackrabbit.oak.plugins.index.lucene.LuceneIndexEditor damassetlucene(` `/oak` `:index` `/damAssetLucene` `) = Indexed 9000 nodes...` |
| --- |


<b>Procédure d’optimisation des index</b>:

1. Accédez à http://*aem-host:port*/crx/de/index.jsp et connectez-vous en tant qu’administrateur
2. Accédez à /oak:index
3. Ajoutez les propriétés mises en évidence ci-dessous 1 aux index correspondants.  Le <b>includedPaths</b> et <b>excludedPaths</b> Les propriétés doivent être une chaîne à plusieurs valeurs ( chaîne )
4. Cliquez sur &quot;[!UICONTROL Enregistrer tout]&quot; dans CRXDe
