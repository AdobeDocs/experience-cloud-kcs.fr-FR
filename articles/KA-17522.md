# Configuration des E/S d’Adobe : Authentification - Étape par étape

## Description

<br>Ce document vous explique comment créer les certificats, clés et jetons à exécuter.<br><br><br><br><br>Objectif<br><br><br><br> <br><br>
Pour fournir des étapes claires et reproductibles afin de générer un jeton porteur d’E/S d’Adobe pour exécuter des appels API pour des solutions DMA telles que Target.


## Résolution

<br><br>Étapes<br><br><br><br> <br><br>
<b>Procédure de création d’un jeton porteur pour exécuter les appels API d’E/S d’Adobe :</b>

1. Génération d’une clé privée et d’un certificat public
2. Création d’une intégration dans la console d’Adobe IO
3. Générer un jeton JWT
4. Échanger le jeton JWT pour un jeton porteur d’accès
5. Jeton du porteur d’accès utilisateur pour exécuter les appels API


- <b>Générez une clé privée et un certificat public selon la documentation suivante :</b> https://www.adobe.io/apis/cloudplatform/console/authentication/createcert.html


   Sur Mac, les commandes suivantes sont intégrées à un terminal. 

   Sur un ordinateur, vous devez télécharger Cygwin (ou tout autre outil de préférences personnelles) et l&#39;exécuter à partir de la ligne de commande. Voici les étapes à suivre pour télécharger et installer Cygwin :

   1. Accédez à https://cygwin.com/install.html
   2. Téléchargez et exécutez setup-x86_64.exe

   <b>*Remarque</b>:  Votre répertoire personnel est le suivant : C:\cygwin64\home\WINDOWSUSER

   Vous pouvez rechercher et installer des packages supplémentaires pendant la phase d’installation. Je vous recommande d’installer tout ce qui concerne &quot;curl&quot; et &quot;ssh&quot;.*


   Voici la commande à exécuter :
   <br> <br> <br>

   | `$ openssl req -nodes -text -x509 -newkey rsa:2048 -keyout secret.pem -out certificate.pem -days 356` |
   | --- |

<br> 

    &lt;b>Remarque&lt;/b> :  Il vous pose plusieurs questions à remplir pour générer le certificat, voir la capture d’écran ci-dessous :
    
![rtaImage](https://helpx.adobe.com/content/dam/help/en/marketing-cloud-core/kb/adobe-io-authentication-step-by-step/jcr%3acontent/main-pars/procedure/proc_par/step_0/step_par/image/rtaImage.png "rtaImage")<br> 

    Une fois les fichiers créés, vous convertissez le fichier secret.key en secret.pem à l’aide de la commande suivante :
    
<br> <br> <br>

    | `$ openssl pkcs8 -topk8 -informez PEM -outform DER -&quot;in` `secret.pem -nocrypt secret.key` |
    | — |
