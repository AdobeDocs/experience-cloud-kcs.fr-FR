# Configuration d’Adobe I/O : Authentification - étape par étape

## Description

<br>Ce document vous explique comment créer les certificats, clés et jetons à exécuter.<br><br><br><br><br>Objectif<br><br><br><br> <br><br>
Pour fournir des étapes claires et reproductibles afin de générer un jeton du porteur Adobe I/O pour exécuter des appels API pour des solutions DMA telles que [!DNL Target].


## Résolution

<br><br>Étapes<br><br><br><br> <br><br>
<b>Procédure de création d’un jeton du porteur pour exécuter les appels API d’Adobe I/O :</b>

1. Générer une clé privée et un certificat public
2. Créer une intégration dans la console Adobe I/O
3. Générer un jeton JWT
4. Échanger le jeton JWT contre un jeton du porteur d’accès
5. Jeton du porteur d’accès utilisateur pour exécuter les appels API


- <b>Générez une clé privée et un certificat public selon la documentation suivante :</b> https://www.adobe.io/apis/cloudplatform/console/authentication/createcert.html


   Sur Mac, les commandes suivantes sont intégrées à un terminal. 

   Sur un ordinateur, vous devez télécharger [!DNL Cygwin] (ou tout autre outil que vous préférez) et l’exécuter à partir de la ligne de commande. Les étapes à télécharger et à installer sont les suivantes : [!DNL Cygwin] :

   1. Accédez à https://cygwin.com/install.html
   2. Télécharger et exécuter `setup-x86_64.exe`.

   <b>*Remarque</b> :  votre répertoire personnel est le suivant : `C:\cygwin64\home\WINDOWSUSER`.

   Vous pouvez rechercher et installer des packages supplémentaires pendant la phase d’installation. Je vous recommande d’installer tout ce qui concerne « `curl` » et « `ssh.` »*.


   Voici la commande à exécuter :
   <br> <br> <br>

   | `$ openssl req -nodes -text -x509 -newkey rsa:2048 -keyout secret.pem -out certificate.pem -days 356` |
   | --- |

<br> 

    &lt;b>Remarque&lt;/b> :  il vous pose plusieurs questions à remplir pour générer le certificat, voir la capture d’écran ci-dessous :
.![rtaImage](https://helpx.adobe.com/content/dam/help/en/marketing-cloud-core/kb/adobe-io-authentication-step-by-step/jcr%3acontent/main-pars/procedure/proc_par/step_0/step_par/image/rtaImage.png "rtaImage")<br> 

    Une fois les fichiers créés, vous convertissez le `secret.key` en `secret.pem` à l’aide de la commande suivante :
<br> <br> <br>

    | `$ openssl pkcs8 -topk8 -inform PEM -outform DER -` `in` `secret.pem  -nocrypt  secret.key` |
    | --- |

<br>

   - Cliquez sur l’icône « Copier » sous « JWT généré ».
