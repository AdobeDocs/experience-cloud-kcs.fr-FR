# Étapes de migration d’une banque de données AEM existante vers AWS S3

## Description

Cet article est lié aux problèmes lors de la migration d’une banque de données AEM existante vers AWS S3.

## Résolution

Vous trouverez ci-dessous les étapes de migration S3 :

1. Activation des journaux de débogage pour les modules `org.apache.jackrabbit.oak.blob` et `org.apache.jackrabbit.oak.plugins.blob`.

1. Fermez AEM.

1. Sauvegardez la configuration FDS existante dans `../crx-quickstart/install/`.

1. Supprimez la variable `FileDataStore.config` in `../crx-quickstart/install/`.

1. Copiez le connecteur S3 dans la fenêtre `../crx-quickstart/install/` dossier.

1. Créez le fichier de configuration S3 dans `../crx-quickstart/install/`. Ajoutez les configurations suivantes :

   ```
   accessKey=
   connectionTimeout="120000"
   maxConnections="40"
   maxErrorRetry="10"
   s3Bucket="XXXXXXXX"
   s3Region="XXXXXXXX"
   secretKey="XXXXXXXX"
   socketTimeout="120000"
   writeThreads="30"
   path="crx-quickstart/repository/datastore"
   ```

1. Dossiers créés - `upload/download/tmp` sous-répertoires à l’intérieur `../crx-quickstart/repository/datastore/`.

1. Assurez-vous que le dossier d’installation et `../crx-quickstart/repository/datastore` Les dossiers sont la propriété de l’utilisateur avec lequel AEM est en cours d’exécution.

1. Déplacer tout le contenu depuis `../crx-quickstart/repository/datastore/` to `../crx-quickstart/repository/datastore/upload/` dossier.

1. Mettez à jour le mode d’exécution dans le script de démarrage afin que AEM commence par `crx3tar-nofds runmode`.

1. Supprimer l’ancien paramètre AEM concernant les modes de démarrage (c’est-à-dire rechercher et supprimer) `sling.options.file`)

1. Supprimer l’ancienne configuration FDS AEM mise en cache de `../crx-quickstart/launchpad/config/org/apache/jackrabbit/oak/plugins/blob/datastore/`.

1. Démarrez AEM.

Étapes suivantes:

1. Surveillez la variable `error.log` lorsque AEM démarre le téléchargement des fichiers dans S3 et que le démarrage est terminé.

1. Vérifiez le dossier de chargement et vérifiez s’il est vide.

1. Vérifiez que tous les lots sont démarrés et qu’il n’existe aucune configuration FileDataStore et qu’il n’existe que la configuration S3.

1. Vérifier si nouveau `sling.options.file` est créée.
